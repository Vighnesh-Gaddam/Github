Git Configuration and Usage

Configuring Git

To configure Git settings, use the following commands:

git config --global user.name "<name>"
git config --global user.email "<email@female.com>"
git config --global core.editor "<editor --wait>"
git config --global core.autocrlf <true|false>

To edit Git configurations:

git config --global -e

Git Installation and Basics

Git Stages

  • U: Untracked
  • A: Added or Staged
  • C: Committed

Common Commands

  • Check current status of files: 
    git status -s
    To know the current status of unstaged and staged files.
  • View commit history: 
    git log --oneline --graph
    To know the current status of saved points (Commits).

Managing Projects with Git

  1. Initialize Git in a project folder: 
    git init
    When you create a new folder, Git does not know about it. You need to initialize Git in the folder to give it permissions. Once initialized, Git can track the folder, and you can create untracked, tracked, modified, staged, and saved checkpoints.
  2. Create Checkpoints (Commits):
    • Add files: 
      git add <file>
    • Stage files: 
      git add .
    • Commit changes: 
      git commit -m "Your commit message"
  3. Go back to a previous commit: 
    git reset --hard HEAD~1
  4. Check file statuses and saved checkpoints:
    • File statuses: 
      git status -s
    • Commit history with graph: 
      git log --oneline --graph
      Check how many files are at which stage and view your saved checkpoints with a visual graph.

Handling Conflicts

Conflicts arise when merging branches with differing changes. Resolve conflicts by choosing between incoming changes (red) and current changes (green), or combining both.

Branches

  • List all branches: 
    git branch
  • Create a new branch: 
    git branch <branchname>
  • Delete a branch: 
    git branch -d <branchname>
  • Switch branches: 
    git switch <branchname>
  • Create and switch to a new branch: 
    git switch -C <branchname>
  • Merge branches: 
    git merge <branchname>

Stashing Changes

When you need to switch branches but have uncommitted changes:

  • Stash changes: 
    git stash
    This command saves your uncommitted changes so you can switch branches without losing them.
  • Apply stashed changes: 
    git stash apply
    This command retrieves the stashed changes and applies them to your working directory.
